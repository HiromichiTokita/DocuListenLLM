# DocuListenLLM 紹介・提案資料
### どんな日本語テキストでも朗読できる、個人開発ソフトウェアの紹介と今後の可能性について

---

## はじめに

本資料は、個人で開発・使用している朗読ソフトウェア「DocuListenLLM」の紹介と、
今後の可能性についてご共有するものです。

元々は**自分が使うために作ったツール**です。
「好きな小説を作業中に聴きたい」というシンプルな動機から開発を始めました。

---

## 1. 何ができるソフトか

### 一言で言うと

**「どんな日本語テキストでも、ボタン一つで朗読できるソフトウェア」**

TXTファイル・EPUB（電子書籍）を読み込んで再生するだけ。
インターネット不要・テキストデータの外部送信なし・完全ローカル動作。

### 何が読めるか

| テキストの種類 | 例 |
|-------------|---|
| 小説・ライトノベル | 購入した電子書籍（EPUB）、テキスト形式の作品 |
| Web小説 | カクヨム・小説家になろう等からコピーしたテキスト |
| 社内文書 | マニュアル・報告書・議事録 |
| 学術・技術文書 | 論文・仕様書・技術ドキュメント |
| 個人メモ | 自分で書いたノート・日記 |

既存の音声書籍サービス（Audible等）は「収録済みのコンテンツしか聴けない」制約がありますが、
本ツールにその制約はありません。

---

## 2. 主な機能

### 音声・再生
- **VOICEVOX**による日本語音声（45以上のキャラクター）
- 話速 0.5x〜3.0x（タイムストレッチ方式）
- 読んだ位置を自動記憶。次回起動時に続きから再生
- BGMファイルを音声に混合して再生
- WAVファイルとして書き出し
- 複数ファイルの連続自動再生

### テキスト処理
- TXT・EPUB（電子書籍）の直接読込
- 青空文庫ルビ（《よみ》形式）の自動かな変換
- 目次・不要テキストの自動スキップ

## 4. 現状の課題・正直なところ

個人開発ツールとして使えてはいますが、製品としてはまだ課題があります。

| 課題 | 現状 |
|------|------|
| 日本語の読みの正確さ | **保証しません。** 人名・固有名詞・専門用語などで誤読が発生します |
| エンジンの動作環境 | VOICEVOXはある程度のスペックが必要。低スペック環境では重い |
| UI/UX | パッケージ配布によりインストールは簡単。ただしUIは改善余地あり |
| AI配役の精度 | 後述の研究中機能。精度にばらつきがある |

日本語の読みについては「だいたい読んでくれる」程度の期待値が適切です。
正確な読みを必要とする用途（ナレーション制作・公式コンテンツ等）には向きません。

## 5. 研究中の機能：AI自動配役

ローカルLLM（Ollama）を使った**AI自動配役機能**を研究中です。
テキスト内の登場人物をAIが分析し、ナレーション・主人公・悪役など
15種類のアーキタイプに自動分類して、それぞれ異なる声で読み分けます。

```
ナレーション：「夕暮れの城壁に、ソーラは一人立っていた。」→ 落ち着いた語り
主人公（女）：「今夜、決着をつける」                    → 若い女性の声
悪役（男）：「……ようやく来たか」                       → 低く重厚な声
```

**現状**：プロトタイプとして動作しますが、配役精度にばらつきがあり、
作品・文体によって精度差が大きい段階です。

---

## 6. 今後こうなれば面白いと思うこと

製品としての可能性を考えると、以下の改善ができれば魅力が増すと思っています。

| 改善点 | 内容 |
|--------|------|
| 日本語読みの精度向上 | 辞書強化・読み仮名エンジンの改善（現状は保証なし） |
| 軽量音声エンジンへの対応 | VOICEVOX以外の軽量TTSエンジンを選べるようにして動作環境の幅を広げる |
| 小さなローカルAIモデルでの配役精度改善 | スペックを問わず動くAI配役の実現 |
| UI のブラッシュアップ | パッケージ配布済みで導入は簡単。操作性のさらなる改善が望ましい |
| スマートフォン・ブラウザ対応 | いつでもどこでも使える環境 |

これらが実現できれば、個人向けアプリや企業向けアクセシビリティツールとしての展開も
考えられるのではないかと思っています。
