# DocuListenLLM 紹介資料
### どんな日本語テキストでも朗読できる、個人開発ソフトウェアの紹介

---

## はじめに

本資料は、個人で開発・使用している朗読ソフトウェア「DocuListenLLM」の紹介です。

元々は**自分が使うために作ったツール**です。
「好きな小説を作業中に聴きたい」というシンプルな動機から開発を始めました。

---

## 0. なぜ作ったか：既存サービスの問題点

きっかけは、既存の朗読・音声化サービスへの不満でした。

**① 音声書籍サービス（Audible 等）の問題**

収録済みのコンテンツしか聴けません。
読みたいものが配信されていなければ、それまでです。

**② クラウドTTS（テキスト読み上げAPI）の問題**

任意のテキストを音声化できますが、**文字数単位の従量課金**のため、
小説1冊（10〜30万字）を読もうとすると費用がいくらかかるか見当がつきません。
プライバシーの観点からも、テキストを外部サーバーに送ることへの懸念があります。

**→ 「収録済み以外のものを、費用を気にせず、ローカルで朗読できるもの」を探し始めたところ、
　　ちょうどよいものが見当たらなかったため、自分で作ることにしました。**

## 1. 何ができるソフトか

### 一言で言うと

**「どんな日本語テキストでも、ボタン一つで朗読できるソフトウェア」**

TXTファイル・EPUB（電子書籍）を読み込んで再生するだけ。
インターネット不要・テキストデータの外部送信なし・完全ローカル動作。

### 何が読めるか

| テキストの種類 | 例 |
|-------------|---|
| 小説・ライトノベル | 購入した電子書籍（EPUB）、テキスト形式の作品 |
| Web小説 | カクヨム・小説家になろう等からコピーしたテキスト |
| 社内文書 | マニュアル・報告書・議事録 |
| 学術・技術文書 | 論文・仕様書・技術ドキュメント |
| 個人メモ | 自分で書いたノート・日記 |

---

## 2. 主な機能

### 音声・再生
- **VOICEVOX**による日本語音声（45以上のキャラクター）
- 話速 0.5x〜3.0x（タイムストレッチ方式）
- 読んだ位置を自動記憶。次回起動時に続きから再生
- WAVファイルとして書き出し
- **プレイリスト**による複数ファイル連続自動再生（再起動後も保持）

### テキスト処理
- TXT・EPUB・PDF・DOCX の直接読込
- ファイルの**ドラッグ＆ドロップ**対応（テキストエリア／プレイリスト両方）
- 青空文庫ルビ（《よみ》形式）の自動かな変換
- 目次・不要テキストの自動スキップ

## 4. 現状の課題・正直なところ

個人開発ツールとして使えてはいますが、製品としてはまだ課題があります。

| 課題 | 現状 |
|------|------|
| 日本語の読みの正確さ | **保証しません。** 人名・固有名詞・専門用語などで誤読が発生します |
| エンジンの動作環境 | VOICEVOXはある程度のスペックが必要。低スペック環境では重い |
| UI/UX | パッケージ配布によりインストールは簡単。ただしUIは改善余地あり |
| AI配役の精度 | 後述の研究中機能。精度にばらつきがある |

日本語の読みについては「だいたい読んでくれる」程度の期待値が適切です。
正確な読みを必要とする用途（ナレーション制作・公式コンテンツ等）には向きません。

## 5. 研究中の機能：AI自動配役

ローカルLLM（Ollama）を使った**AI自動配役機能**を研究中です。
テキスト内の登場人物をAIが分析し、ナレーション・主人公・悪役など
15種類のアーキタイプに自動分類して、それぞれ異なる声で読み分けます。

```
ナレーション：「夕暮れの城壁に、ソーラは一人立っていた。」→ 落ち着いた語り
主人公（女）：「今夜、決着をつける」                    → 若い女性の声
悪役（男）：「……ようやく来たか」                       → 低く重厚な声
```

操作フローは「📖自動抽出」→「🤖AI配役」→「▶再生」の3ステップ。
処理はセリフをまとめてLLMに投げるバッチ方式で、500セリフを約50秒で処理します。
Ollamaの起動確認・モデル確認もワンクリックで行えます。

**現状**：動作しますが、配役精度にばらつきがあり、作品・文体によって精度差が大きい段階です。
完全な自動化よりも「大まかに配役してから手動で微調整する」用途での利用を想定しています。
