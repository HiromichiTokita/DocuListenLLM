# DocuListenLLM — プロジェクト指示

## 引継ぎ資料の更新ルール（最重要・必須）

**何らかの進捗があったら、必ず `handoff.md` を更新すること。** 例外なし。

- 対象となる「進捗」: コード変更（機能追加/修正/削除）、依存・ビルド設定の変更、
  バージョン変更、設計・計画の決定、サブプロジェクト（IrodoriTTS等）の状態変化、
  既知の課題/TODO の増減、配布物の更新 など。
- 更新内容: 該当セクション（履歴・TODO・サブPJ状態・重要ステータス等）に、
  **日付（YYYY-MM-DD）付き**で何をしたかを追記/修正する。完了したTODOは `[x]` 取消線にする。
- タイミング: その作業を終えた直後（コミット相当の区切り）に更新する。後回しにしない。
- `handoff.md` はこのプロジェクトの**唯一の引継ぎ資料**。旧 `HANDOVER.md` /
  `DEVELOPMENT_HISTORY.md` は統合・廃止済み。新たな履歴系ファイルを増やさない。

## 環境メモ

- 本体の実行/ビルドは **`py`（=Python 3.14）** を使う。PATH の `python` は 3.12 で別物。
- 本体リポジトリは **Git 管理外**＝ファイル削除は復元不可。削除前に必ず確認する。
- IrodoriTTS サーバの作業先は `E:\project\Irodori-TTS-code`（別venv・別プロセス・別git）。
